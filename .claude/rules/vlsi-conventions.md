# Verilog Coding Conventions

## Signal Naming
- Use snake_case for all signal names
- Active-low signals end with _n (e.g., rst_n, valid_n)
- Clock signals: clk, clk_<domain> (e.g., clk_axi, clk_core)
- Reset signals: rst_n, rst_<domain>_n

## Port Declarations
- Group by direction: inputs first, then outputs, then inouts
- Declare width explicitly: input [31:0] data_in
- Single-bit signals: input valid (not input [0:0] valid)

## Always Blocks
- Use non-blocking assignments (<=) for sequential logic
- Use blocking assignments (=) for combinational logic
- One always block per clock domain
- Always include reset in sequential blocks

## Testbench Conventions
- Module name: tb_<module_under_test>
- Clock period: 10ns (100MHz) unless specified otherwise
- Reset: active-low, hold for 5 clock cycles minimum
- Use $display for messages, not $write
- Use $finish to end simulation, not $stop
- Include pass/fail status in output

## Comments
- Add // GENERATED BY VLSI-SUPER-AGENT at top of generated files
- Comment complex logic, not obvious assignments
- Use // for single-line, /* */ for multi-line

## Vivado xsim Compatibility
- No SystemVerilog features unless RTL uses them
- Use $urandom_range not $random
- All signals must be declared (no implicit wires)
- No fork-join, no UVM constructs
